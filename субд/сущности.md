> а какие вобще правила касательно того, что хранить в сущности - id или объект?

Вот концептуальные правила, которыми обычно руководствуются:

------

### 1️⃣ Доменная сущность хранит **id**, если:

- связь с другой сущностью нужна **только для идентификации**.
- нет необходимости работать с объектом напрямую внутри бизнес-логики.
- хотим избежать сильной связанности (loose coupling).

Пример: `CharacterEntity` хранит `projectId`, а не `ProjectEntity`.

------

### 2️⃣ Хранить объект другой сущности в сущности, если:

- сущность действительно **зависит от объекта целиком** для своей логики.
- нужны методы или инварианты, которые используют поля связанной сущности.
- связь **агрегатная**, и ты хочешь гарантировать инварианты через агрегат.

Пример: у `Order` может быть объект `Customer` внутри, если логика заказа реально зависит от данных клиента.

------

### 3️⃣ Практическое правило

- **Для внешних ключей и простых связей — храни только id.**
- **Для агрегатов и инвариантов — можно хранить объект.**
- Излишнее хранение объектов → рост связанности, сложности с маппингом и сериализацией.

------

Если хочешь, могу на твоём примере персонажа и проекта нарисовать **когда id, а когда объект**, чтобы было совсем наглядно.

> Грубо говоря, если есть сущность "Посылка" и у нее есть правило "суммарный вес не должен быть больше 10кг", тогда в поле "items" надо хранить не ссылки на вещи, а сами вещи? типа потому что у каждой вещи есть вес и инвариант посылки можно предохранить только если мы этот вес можем реально проссумировать, а для этого одних только id не достаточно?

Да, именно так.

Если **инвариант агрегата зависит от данных дочерних объектов**, то эти объекты нужно хранить **внутри агрегата**, а не только их id.

В твоём примере:

```
class Package {
  private items: Item[]; // сами объекты, чтобы можно было суммировать вес

  addItem(item: Item) {
    if (this.totalWeight() + item.weight > 10_000) {
      throw new Error("Суммарный вес не должен превышать 10 кг");
    }
    this.items.push(item);
  }

  private totalWeight() {
    return this.items.reduce((sum, i) => sum + i.weight, 0);
  }
}
```

- `items` как id не хватило бы для проверки инварианта.
- Если бы вес проверялся на уровне БД или вручную в usecase, это уже не гарантия доменной целостности.

**Правило:** храним объекты в сущности **только когда они нужны для соблюдения инвариантов агрегата**.
