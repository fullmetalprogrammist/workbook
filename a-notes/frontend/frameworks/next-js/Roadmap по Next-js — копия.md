### **Roadmap по освоению Next.js (версия 14+)**

---

### **Этап 1: Основы Next.js (Стать продуктивным)**

**Цель:** Понимать философию Next.js и уметь делать простые сайты.
1.  **App Router vs Pages Router:** Понимание разницы. **Фокус на App Router** (актуально с версии 13+).
2.  **Структура проекта (App Router):**
    *   Роутинг на основе папок (`app/`).
    *   Роли файлов: `page.js` (страница), `layout.js` (общий макет), `loading.js` (загрузка), `error.js` (ошибка), `not-found.js` (404).
    *   Группировка роутов с помощью папок `(auth)` и параллельных слотов `@sidebar`.
3.  **Навигация и Линковка:**
    *   Компонент `<Link>` для SPA-переходов.
    *   Хук `useRouter()` для програмной навигации.
    *   Префетчинг данных при наведении.
4.  **Рендеринг:**
    *   **React Server Components (RSC):** Компоненты по умолчанию. Как и когда использовать.
    *   **Клиентские компоненты:** Директива `'use client'`. Когда нужна интерактивность (хуки, события).
    *   Практика: Разделите простую страницу на серверную и клиентскую части.
5.  **Работа с данными (Data Fetching):**
    *   Использование нативного `fetch()` в Server Components.
    *   Кэширование и revalidation: `cache: 'force-cache'` (SSG), `cache: 'no-store'` (SSR), `next: { revalidate: 60 }` (ISR).
6.  **Стилизация:**
    *   Next.js способ: `CSS Modules` (импорт `styles from './Component.module.css'`).
    *   Популярные опции: `Tailwind CSS`, `styled-components` (требует специальной настройки).
7.  **Метаданные и SEO:**
    *   Статический экспорт: `generateMetadata` функция.
    *   Динамический экспорт: `generateMetadata` с параметрами.
8.  **Развертывание (Deploy):**
    *   **Vercel:** Бесплатный хостинг от создателей Next.js (проще всего).
    *   Другие опции: Netlify, AWS, собственный сервер.

---

### **Этап 2: Продвинутый уровень (Углубление и Оптимизация)**

**Цель:** Делать сложные, быстрые и безопасные приложения.
1.  **Оптимизации из коробки:**
    *   **Компонент `<Image>`:** Оптимизация, ресайз и lazy loading изображений.
    *   **Компонент `<Script>`:** Контроль загрузки сторонних скриптов.
    *   **Фонты:** Автоматическая оптимизация шрифтов с `next/font`.
2.  **Маршрутизация:**
    *   **Динамические роуты:** `app/blog/[slug]/page.js`.
    *   **Перехват роутов:** `app/blog/(.)[slug]/page.js` (для модальных окон).
    *   **Middleware:** Функции, выполняемые до обработки запроса. Для аутентификации, логирования, редиректов, работы с заголовками.
3.  **Углубленная работа с данными:**
    *   **Обработка ошибок и загрузок:** Использование `error.js` и `loading.js` для изоляции состояний.
    *   **Parallel Data Fetching:** Оптимизация запросов путем их одновременного запуска.
    *   **Библиотеки для кэширования:** `TanStack Query (React Query)` для сложного клиентского кэша.
4.  **Формы и Действия (Actions):**
    *   **Server Actions:** Инновация Next.js. Вызов функций на сервере прямо из React-компонента (без создания отдельного API endpoint). Для форм, лайков, комментариев.
5.  **API Routes:** Создание собственного API внутри Next.js-приложения (`app/api/route.js`).
6.  **Аутентификация:**
    *   Обзор паттернов: `Next-Auth` (популярное решение), `Clerk`, `Supabase Auth`.
    *   Интеграция аутентификации в Middleware для защиты роутов.
7.  **Состояние приложения (State Management):**
    *   **Серверное состояние:** `TanStack Query` (лучше для данных с бэкенда).
    *   **Клиентское состояние:** `Zustand` (легче Redux), `useContext` + `useReducer`.
8.  **Тестирование:** Настройка и написание unit-тестов (`Jest`, `React Testing Library`) и e2e-тестов (`Playwright`, `Cypress`).

---

### **Этап 3: Экспертный уровень (Архитектура и Производительность)**

**Цель:** Принимать архитектурные решения, максимально ускорять и масштабировать приложения.
1.  **Углубленное понимание RSC:**
    *   Потоки данных (Streaming) и `Suspense`.
    *   Постепенная миграция на RSC в legacy-проектах.
2.  **Анализ и Оптимизация бандла:**
    *   Анализ: `next bundle-analyzer`.
    *   Динамический импорт: `next/dynamic` для ленивой подгрузки тяжелых клиентских компонентов.
3.  **Кэширование и Performance Tweaking:**
    *   Глубокое понимание кэширующих механизмов Next.js (Router Cache, Full Route Cache, Request Memoization).
    *   Настройка `next.config.js` для сложных сценариев.
4.  **Интернационализация (i18n):** Поддержка нескольких языков.
5.  **Паттерны и Антипаттерны:** Лучшие практики для построения масштабируемых приложений на Next.js.
6.  **Монолиты и Micro Frontends:** Использование Next.js в составе большой архитектуры.
