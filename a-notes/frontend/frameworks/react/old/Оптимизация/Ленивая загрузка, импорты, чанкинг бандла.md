# Бандл и чанки

Когда браузер открывает SPA-приложение, он сначала получает пустой html с root-элементом, а потом начинает загружать js-бандл. Если бандл единый, т.е. все приложение оформлено в виде единственного js-файла, то первичная загрузка может быть долгой.

Для ускорения загрузки используется чанкинг - разбиение бандла на несколько js-файлов (чанков), чтобы сперва загрузилось только то, что нужно в первую очередь, а потом все остальное. Базовый пример - в приложении 4 страницы, каждую можно оформить отдельным чанком. Пользователь сходу загрузит только чанк главной страницы, а если не захочет переходить на другие страницы, то их чанки не загрузятся.

Разбиение можно сделать:

- Автоматически
- Вручную

## Автоматический чанкинг

Зависит от того, как мы импортируем компоненты.

При статическом импорте

```jsx
import SomeComponent from './SomeComponent';
```

код компонента помещается в основной бандл.

При динамическом импорте

```jsx
const LazyLoadedComponent = React.lazy(() => import('./LazyLoadedComponent'));
```

сборщик при сборке приложения выделяет код компонента в отдельный чанк (js-файл).

## Ручной чанкинг

Делается через хинты при динамическом импорте:

```javascript
// Webpack
// Страницы будут в одном чанке с именем "pages"
const Home = React.lazy(() => import(/* webpackChunkName: "pages" */ './pages/Home'));
const About = React.lazy(() => import(/* webpackChunkName: "pages" */ './pages/About'));
```

```javascript
// Vite
const Home = React.lazy(() => import(/* webpackIgnore: true */ './pages/Home?chunk=pages'));
const About = React.lazy(() => import(/* webpackIgnore: true */ './pages/About?chunk=pages'));
```

Хинты зависят от сборщика, универсального решения нет.



# Как применять ленивую загрузку

## React.lazy, Suspect, ErrorBoundary

```jsx
import React, { Suspense } from 'react';
import { BrowserRouter as Router, Routes, Route } from 'react-router-dom';
import ErrorBoundary from './foo/bar/ErrorBoundary';

const Home = React.lazy(() => import('./pages/Home'));
const About = React.lazy(() => import('./pages/About'));
const Contact = React.lazy(() => import('./pages/Contact'));

function App() {
  return (
    <Router>
      <ErrorBoundary>
        <Suspense fallback={<div>Загрузка страницы...</div>}>
          <Routes>
            <Route path="/" element={<Home />} />
            <Route path="/about" element={<About />} />
            <Route path="/contact" element={<Contact />} />
          </Routes>
        </Suspense>
      </ErrorBoundary>
    </Router>
  );
}
```

Ленивая загрузка ErrorBoundary > Suspense > React.lazy > import()

- Сборщик при сборке приложения видит динамический импорт Home, About, Contact и делает из них отдельные чанки.
- import - это функция, которая возвращает промис. Он резолвится, когда модуль загружен. Здесь вызов import выполнен внутри лямбды, а она передана в React.lazy. Получился "ленивый компонент" - при попытке его отрендерить вызывается лямбда, т.е. выполняется динамический импорт и браузер начинает загружать чанк.
- Suspence перехватывает промис и пока он не завершился, отображает fallback-элемент.
- Когда чанк загрузился, fallback заменяется на компонент.
- ErrorBoundary ловит ошибку в случае если чанк не загрузился и показывает fallback.
  - Если промис из import отклоняется, React.lazy выбрасывает исключение вместо рендера компонента. ErrorBoundary ловит это исключение и показывает свой fallback.
  - ErrorBoundary - это паттерн. Фактически это самописный компонент, в котором мы можем залогировать ошибку и вернуть fallback-разметку, которую надо показать вместо упавшего компонента.
  - ErrorBoundary можно реализовать только на классовых компонентах.

# Предзагрузка

- Плюс предзагрузки в том, что если чанк успеет загрузиться к моменту рендера ленивого компонента, то компонент отобразится мгновенно, без fallback.
- import - умная функция. Она возвращает один и тот же промис при повторных запросах одного и того же модуля. Поэтому если чанк не успеет загрузиться по предзагрузке, то рендер ленивого компонента получит уже созданный промис.

```jsx
import React, { useState, Suspense } from 'react';

// Объявляем ленивый компонент
const LazyComponent = React.lazy(() => import('./LazyComponent'));

// Функция для предзагрузки компонента
const preloadLazyComponent = () => {
  import('./LazyComponent');
};

function MyComponent() {
  const [showComponent, setShowComponent] = useState(false);

  const handleClick = () => {
    setShowComponent(true);
  };

  return (
    <div>
      <button
        onMouseEnter={preloadLazyComponent} // Предзагрузка при наведении
        onClick={handleClick} // Показ компонента по клику
      >
        Показать компонент
      </button>

      {showComponent && (
        <Suspense fallback={<div>Загрузка...</div>}>
          <LazyComponent />
        </Suspense>
      )}
    </div>
  );
}

export default MyComponent;
```
