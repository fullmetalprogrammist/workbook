# Переменные окружения

## Зачем нужны

Переменные окружения - это переменные, которые находятся в памяти запущенного процесса. Значение для них задается при старте процесса и остается неизменным, пока процесс запущен. Чтобы изменить значения, надо остановить процесс и снова запустить его, указав новые значения.

Получить к ним доступ возможно только если есть непосредственный доступ к ОС сервера. Т.е. надо удаленно зайти на сервер как администратор и через команду ОС посмотреть. Иначе - через браузер например - невозможно.

Поэтому переменные окружения считаются безопасным способом хранения чувствительных данных, вроде логина и пароля для подключения к БД.

## Как пользоваться на практике

При локальной разработке в корне приложения создаются файлы вроде .env.local, в них пишутся имена и значения переменных. На старте приложения эти значения считываем, обычно фреймворки делают это самостоятельно, например Next.js и предоставляют удобные средства доступа к этим переменным из программы. Эти .env-файлы надо не забывать добавлять в gitignore, чтобы секреты не утекли.

После деплоя приложения на хостинг значения переменных окружения можно задать через интерфейс хостинга. Он сохранит их в зашифрованном виде в свою БД и будет автоматически подставлять при запуске нашего приложения.

## Подводные камни

В классических SPA-приложениях переменные окружения для хранения секретов не используются. Т.к. приложение выполняется у пользователя в браузере, то единственный способ снабдить его значениями - это зашить их в бандл с кодом приложения. В этом случае секреты можно найти или в самом бандле, или просто посмотреть в DevTools браузера в разделе "Сеть" - значения секретов будут подставляться в запрос и их будет отлично видно.

Поэтому для таких случаев либо делается BFF (backend-for-frontend), либо используются фреймворки вроде Next.js, которые имеют особую архитектуру, совмещающую в себе как бы SPA + серверную часть.
