# Локальная установка kubernetes

- Ссылка на официальный мануал
  - https://kubernetes.io/ru/docs/tasks/tools/install-kubectl/
- Установка на Windows 10 с использованием встроенного в нее гипервизора Hyper-V

## Windows

* Для работы нам понадобится гипервизор.
  
* В Windows 10 он встроен, но мб отключен.
  
  * Как включить
    
    * Открываем PowerShell от имени администратора.
    
    * Проверяем, не включен ли он уже
    
      ```
      Get-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V
      ```
    
      Смотрим пункт `State`. Если Disabled, надо включить.
    
    * Для включения выполняем команду
    
      ````
      Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
      ````
    
    * Перезагружаем компьютер
  
* Качаем kubectl

  * Это консольная утилита для взаимодействия с кластером кубера
  * https://dl.k8s.io/release/v1.35.0/bin/windows/amd64/kubectl.exe
    * Вот тут https://dl.k8s.io/release/stable.txt можно узнать актуальную версию и поставить ее в ссылку выше
  * Кладем его в директорию, прописанную в PATH
    * Например, создадим директорию `C:\other\k8s` и положим туда скачанный `kubectl.exe`
    * Добавим эту директорию в PATH
      * *Пуск > Параметры > О программе > Дополнительные параметры системы > Переменные среды > Системные переменные > Path > Изменить*
    * P.S. Это может сломать докер, потому что он тоже ставит kubectl. В этом случае можно просто удалить нашу запись.
    * Чтобы работала наша версия, а не докерская, наша должна стоять в path выше докерской. Например, если строчка докера `C:\Program Files\Docker\Docker\resources\bin`, то нашу добавляем выше нее
    * Запускаем консоль, набираем `kubectl version --client` чтобы убедиться что работает

* Качаем minikube
  
  * https://github.com/kubernetes/minikube/releases/latest/download/minikube-installer.exe
  * Устанавливаем как обычное настольное приложение.
  * Открываем PowerShell от имени администратора
    * Вообще все надо делать из PowerShell (в обычной консоли русские буквы не отображаются) и из-под админа (иначе команды не сработают)
  * Запускаем кластер командой `minikube start --driver=hyperv`
    * Для остановки набираем `minikube stop`
    * Для повторного старта достаточно просто `minikube start` без указания драйвера
    * Если не заморачиваться, то работа происходит с одним и тем же кластером. Но можно использовать и разные кластеры. Как и зачем - это отдельная тема.
  * Проверяем, работает ли kubectl.
    * После установки minikube должен автоматически создаться конфиг для kubectl.
    * Запускаем PowerShell от имени администратора.
    * При выполнении `kubectl cluster-info` должны показаться адреса Control plane и CoreDNS.

## Админка

- После успешного старта всего и вся, командой `minikube dashboard` можно запустить админку, которая автоматически откроется в дефолтном браузере
  - или `minikube dashboard --url`, чтобы просто получить url админки и открыть его где хочешь
- Терминал блокируется, чтобы остановить админку, `Ctrl + C`
