# minikube, кубер, движок контейнеризации

- как все это связано
  - minikube - это просто программа, которая работает у меня на компе
  - она создает кластер кубера из одной ноды
    - эта нода совмещает в себе функции и мозговой ноды и рабочей ноды
  - как именно создается нода, зависит от драйвера
    - примеры драйверов
      - hyperv
      - docker
  - если драйвер hyperv
    - когда мы даем команду запуска  кластера
      - миникуб поднимает пустую виртуалку
      - скачивает из своих источников образ линукса и ставит на эту виртуалку
      - скачивает движок контейнеризации (`containerd` по умолчанию) и все куберские программы
      - устанавливает это все в виртуалку и все, кластер из одной ноды готов к работе
    - чтобы крутить контейнеры в кубере, надо прокинуть их образы в виртуалку
  - если драйвер docker
    - здесь вместо виртуалки используется докер-контейнер, поэтому на компе должен быть установлен докер
    - когда мы даем команду запуска  кластера
      - миникуб поднимает контейнер из `kic` (Kubernetes in container) образа, внутри которого есть движок containerd и куберские проги
      - т.о. докер используется только чтобы крутить этот контейнер, внутри которого по сути работает другой движок контейнеризации и именно он будет крутить образы наших микросервисов
    - крутить контейнеры в кубере в этом случае проще, т.к. есть команды, с помощью которых кубер увидит локальные образы докера
