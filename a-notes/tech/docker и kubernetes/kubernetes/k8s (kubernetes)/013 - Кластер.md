# Кластер

- кластер - это понятие концептуальное, а не "физическое"
- есть много конкретных компьютеров \ конкретных виртуальных машин, они объединены в кластер
  - т.е. получается что кластер - это не какой-то конкретный комп \ виртуалка, а именно логическая совокупность компов \ виртуалок
  - аналогия - общество. Есть множество конкретных людей, про которое мы говорим - "общество"
- комп \ виртуалка внутри кластера называется "нода"
- концептуально в любом кластере (не только в кубере) ноды делятся на две роли
  - управляющие (мозг кластера, управление его работой, в кубере называются `control plane`)
    - обычно делают 3-5 "мозговых" нод для отказоустойчивости
  - рабочие (непосредственно выполняющие работу, `worker nodes`)
- управляющие ноды распределяют работу по рабочим
- обе группы нод представлены обычно несколькими компами \ виртуалками
  - т.е. "мозг" состоит из нескольких компов \ виртуалок, работающих вместе
- как формируется кластер
  - сначала формируется "мозг" - на несколько виртуалок ставится "управляющее" кластером ПО
  - далее на любой из этих виртуалок ("инициатор") запускаем команду инициализации кластера
    - получаем технические данные, которые нужны для присоединения остальных нод
  - далее со всех остальных виртуалок запускаем команду присоединения к кластеру, указывая адрес "инициатора" и флаг, что присоединяем мозговую ноду
    - но как только в кластере получается больше одной "мозговой" ноды, то они становятся равнозначными и присоединять новые ноды можно уже через любую из них, не обязательно через "инициатора"
  - рабочие ноды присоединяются по такому же принципу - запускаем команду присоединения, указывая любую мозговую ноду кластера и указываем в этой команде флаг, что присоединяем рабочую ноду
- для кубера все примерно так и работает на понятийном уровне
  - вероятно и для других кластерных систем не сильно отличается
