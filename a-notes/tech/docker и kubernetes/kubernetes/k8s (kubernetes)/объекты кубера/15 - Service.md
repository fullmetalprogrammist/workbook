



# Service

- type
  - ClusterIP - для организации стабильного доступа к поду
    - доступ только внутри кластера
  - NodePort
  - LoadBalancer - для организации физической точки входа в кластер кубера
    - запрос попадает на сервис этого типа, а из него уже на ingress controller
  - ExternalName
