# Ingress и Ingress Controller

- `ingress`
  - ингрес - это ресурс кубера, который описывает правила маршрутизации HTTP/HTTPS-трафика извне кластера к сервисам внутри кластера
  - ингрес представляет собой yaml-файл, в котором написано, какой маршрут на какой сервис ведет
- `ingress controller`
  - ингрес контроллер - это процесс, который эти правила фактически реализует в жизнь
  - сам по себе контроллер - это как интерфейс, дефолтной реализации которого в кубере нет
    - но есть множество реализаций от разных вендоров
  - контроллер следит за ресурсами кубера и когда они изменяются, он эти изменения видит и применяет
  - по спецификации кубера, контроллер обязан делать только маршрутизацию
    - но вендоры обычно добавляют в свои реализации контроллера дополнительную функциональность вроде rate limiting, авторизацию и т.д.
- в общем, ингрес - это декларация правил, контроллер - их реализация
- ингрес и ингрес-контроллер - это чисто куберовские понятия, вне экосистемы кубера они не существуют
- контроллер помогает интегрировать API Gateway в кластер кубера
  - например
    - Kong - это инструмент, реализующий API Gateway. Может работать вообще без кубера.
    - KIC - Kong Ingress Controller - это реализация ингрес-контроллера, которая берет ингрес-правила и трансформирует их в понятный конгу вид. Т.о. эта реализация контроллера служит для интеграции конга и кубера.
