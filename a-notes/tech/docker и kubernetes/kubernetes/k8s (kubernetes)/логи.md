# Как хранятся логи

- Под каждый под создается директория `var\logs\pods\под\контейнер\файл`
  - Эта директория находится на самой ноде кластера
- Согласно стандарту, все логи помещаются по такому типовому пути, поэтому читаются по одному принципу



# Команды чтения логов

## Если в поде только один контейнер

```
kubectl logs <имя-пода> -n <неймспейс>
```

```
 kubectl logs deployment/traefik -n traefik
```

- Если в поде крутится несколько контейнеров, будет ошибка, т.к. кубер не будет знать, чей лог выводить

## Если в поде несколько контейнеров

- Вывести кашу из логов всех контейнеров - флаг `--all-containers`

```
 kubectl logs deployment/traefik -n traefik --all-containers
```
