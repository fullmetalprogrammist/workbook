# Команды

- все выполняется из PowerShell под администратором
- `kubectl` - команда управления ресурсами внутри кластера (поды, логи и т.д.)
  - `ctl` - в unix это стандартный суффикс для управляющих утилит, означает "control"
    - т.о. kubectl - это "управление кубером"



# Контекст, profile и кластер

- при создании нового кластера
  - создается профиль
  - поднимается кластер
  - создается контекст
    - по умолчанию, имя контекста такое же, как имя профиля кластера
      - технически, они не обязаны совпадать, но best practice - чтобы совпадали
  - этот контекст становится активным
- профиль - это настройки самого кластера (драйвер, память, cpu и т.д.)
- контекст - это набор настроек для доступа к кластеру (адрес кластера, юзер, namespace и т.д.)
- ⚠️ команды от kubectl уходят на кластер, контекст которого сейчас активен



# Работа с контекстом

## Список контекстов

```
kubectl config get-contexts
```

- в списке выведутся только контексты запущенных кластеров
  - т.е. если какой-то кластер не запущен, список не покажет его контекст

## Сделать контекст активным

```
kubectl config use-context имя-контекста
```

- `имя-контекста` - по дефолту совпадает с профилем из колонки profile (minikube profile list)
