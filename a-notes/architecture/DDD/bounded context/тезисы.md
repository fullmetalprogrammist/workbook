# Тезисы

- `Болтливость` ограниченного контекста
  - Это когда контекст не может завершить задачу без обращения к другим контекстам
  - Если это происходит очень часто, вероятно надо пересмотреть границу контекста, чтобы он стал более автономным, самостоятельным
- `Непреднамеренная сложность`
  - Это когда контекст накапливает слишком много разнородной логики, решает много разных задач
  - Это повод разбить контекст на несколько контекстов, чтобы каждый занимался своей задачей
- Ограниченные контексты вместо одной общей модели на всю предметную область позволяют создать несколько более компактных моделей, каждая из которых реализует самостоятельный блок функциональности
  - `Непротиворечивость единого языка`. За счет ограниченного контекста не происходит путаницы в языке. В каждой модели могут быть одинаково звучащие термины, например "Товар", но в каждом контексте он имеет свои собственные характеристики, нужные только в этом контексте, и не захламляется характеристиками из другого контекста. В случае же одной общей модели "Товар" был бы вынужден держать все характеристики разом.
- Размер контекста - трейдофф (компромис)
  - Маленькие контексты - сложнее интеграция
  - Большие контексты - сложнее сохранять непротиворечивость языка
  - Базовое правило - избегать разбиения связанной функциональности на разные контексты
