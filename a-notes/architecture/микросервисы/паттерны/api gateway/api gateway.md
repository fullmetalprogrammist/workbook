# Факты о gateway'ях

- гейт - единственная точка входа в систему
- логический гейт единственный на всю систему
  - реплик может быть много для отказоустойчивости, но логически гейт один
  - много гейтов приводили бы к фрагментации логики маршрутизации, защиты и т.д.



# Техническая часть

- реализация гейта сильно зависит от инфраструктуры
  - в докере - одно, в кубере - другое

## Авторизация

- jwt расшифровывается на гейте
- гейт добавляет данные из jwt в заголовки запросов и отправляет запросы сервисам
- т.о. сервисам не приходится тратить время на расшифровку jwt, они получают уже готовые креды через заголовки





# Терминология

## Типы трафика

- `Север-юг (north-south)` - трафик из внешки в систему
- `Восток-запад (east-west)` - трафик между сервисами внутри системы
- Термины используются в документации кубера и в принципе являются устоявшимися



## Уровни балансировки \ маршрутизации

- `L4` - транспортный уровень
  - работает с адресами и портами, TCP \ UDP, не умеет анализировать url
- `L7` - прикладной уровень
  - работает с HTTP, gRPC и т.д., умеет анализировать url



## Утилитарные вещи

- `sidecar-контейнер`
  - 
  - sidecar переводится как "коляска" (у мотоцикла), названо так потому что коляска добавляет функциональность мотоциклу, не изменяя его конструкцию. Так же и дополнительный контейнер не изменяет функциональность основного сервиса, но добавляет ему функциональность.
