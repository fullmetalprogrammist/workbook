# record

- появились в Java 16 (2021 год)
- компилируется в класс
- зачем нужны
  - удобный способ создать иммутабельный объект без необходимости писать руками бойлерплейт
    - автоматически генерируются
      - методы toString, hashCode, equals
      - методы доступа к полям (называются `accessor methods`)
      - конструктор под все аргументы (в рамках темы record он называется `канонический`)
- канонический конструктор поддерживает синтаксис `compact constructor`
  - такая фишка есть только у record, у обычных классов такого нет
  - такой конструктор автоматически получает все аргументы и после выполнения своего тела автоматически делает присваивание
- особенности record
  - можно:
    - добавлять методы обычные и static
    - добавлять поля (только static)
    - переопределять методы (и обвязочные, и аксессоры)
    - реализовывать интерфейсы
    - перегружать конструктор (с обязательным вызовом канонического)
  - нельзя:
    - наследоваться (ни record ни от кого не может, и никто от record'а не может)
      - каждый record неявно наследуется от `java.lang.Record`
    - изменять значения полей (они private final)
