

# DHCP

- DHCP - Dynamic Host Configuration Protocol
- зачем нужен (концептуально и грубо)
  - когда компьютер без ip-адреса подключается к сети, он кидает в нее широковещательное сообщение `DHCPDISCOVER`
  - когда это сообщение улавливает `DHCP-сервер`, между ним и компьютером начинается процесс согласования (состоит из нескольких шагов), в результате которого компьютер получает IP-адрес
    - а также и другие сетевые настройки
      - маска подсети
      - IP дефолтного шлюза
      - адрес хотя бы одного DNS
      - разные другие штуки
- в домашней сети роль DHCP выполняет роутер ("маршрутизатор")

## ip-адрес и маска подсети, что и зачем

- ip-адрес v4 состоит из 32-х бит
  - например `192.168.20.5`
  - этот адрес содержит две информации - `адрес подсети`, и `адрес устройства` в этой подсети, т.е. два в одном
- маска подсети задает, какая часть ip-адреса относится к подсети, а какая - к устройству
  - маска подсети по длине соответствует длине ip-адреса, если адрес 32 бита, то и маска 32 бита
  - маска представляет собой непрерывную последовательность единиц, потом нулей
    - грубо говоря, маска может быть 11110000, но не может 1010100
  - количество единиц бывает указывают в конце адреса, например `192.168.20.5/20`
  - пример маски `255.255.240.0`
  - чтобы узнать адрес подсети, надо сделать "И" между ip-адресом и маской
    - а чтобы узнать адрес устройства, взять биты ip-адреса, под которыми в маске стоят нули

```
192.168.20.5 : ip
11000000.10101000.00010100.00000101  : ip в двоичном виде
11111111.11111111.11110000.00000000  : маска в двоичном (255.255.240.0)
11000000.10101000.0001----.--------  : адрес подсети (192.168.16.0)
----------------------0100.00000101  : адрес устройства (0.0.4.5)
```
