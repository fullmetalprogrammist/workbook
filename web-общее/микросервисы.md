- api gateway
  - точка входа в "зону микросервисов"
  - принимает запрос от пользователя по http, проверяет авторизацию и принимает решение, в какой сервис отправить запрос
  - т.о. служит для изоляции внешнего мира от системы, никому не известна за счет этого структура системы
- способы общения между микросервисами
  - gRPC
    - используется для случаев, когда сервисам надо общаться с синхронным получением ответа
    - концептуально выглядит так, будто просто в обычном коде вызывается обычная функция
      - технический слой удаленного вызова инкапсулирован в технологии
    - пример - операция связана с оплатой, уходит запрос в сервис оплаты, надо получить синхронный ответ, прошла ли оплата
  - очереди (kafka, rabbitMq)
    - используется, когда синхронный ответ не обязателен, можно и подождать
      - разные уведомления
    - например, "оплата прошла успешно, покупка придет вам на почту"
