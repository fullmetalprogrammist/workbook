  - [ ] что хранить
    - [ ] результаты тяжелых вычислений
    - [ ] часто запрашиваемые данные
- [ ] CDN
- [ ] cache tier
  - [ ] отдельный кэш-тир позволяет масштабировать кэш независимо
- [ ] read-through cache
  - [ ] стратегия работы с кэшем
  - [ ] web сервер сначала ищет нужные данные в кэше
  - [ ] есть - возвращает
  - [ ] нету - берет из БД, кладет в кэш, и возвращает

  - [ ] читаешь часто, пишешь - нечасто
- [ ] expiration policy (политика срока действия)

  - [ ] FIFO
- [ ] синхронизация кэша и БД
