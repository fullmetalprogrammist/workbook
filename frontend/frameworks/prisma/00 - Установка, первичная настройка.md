# Установка  пакетов

```
npm install @prisma/client @prisma/adapter-pg pg dotenv
```

ссылка на [мануал](https://www.prisma.io/docs/getting-started/prisma-orm/quickstart/postgresql#2-install-required-dependencies)

- `@prisma/client` - The Prisma Client library for querying your database
- `@prisma/adapter-pg` - The [`node-postgres` driver adapter](https://www.prisma.io/docs/orm/overview/databases/postgresql#using-the-node-postgres-driver) that connects Prisma Client to your database
- `pg` - The node-postgres database driver



# Инициализация

```
npx prisma init
```

- создает это
  - `.env` - файл, в котором надо будет задать url до БД
  - `prisma.config.ts` - файл в корне проекта, можно в нем ничего не трогать
  - `prisma/schema.prisma` - папка в корне проекта с файлом под описание миграций
    - в этом файле можно изменить настройку `output`, чтобы сгенерированные призмой файлы попадали не черти куда, а по ожидаемому пути
      - это файлы с клиентом призмы, плюс классы модели таблиц и прочая инфраструктура
      - указываемый путь должен существовать, т.е. если его нет, надо создать руками



# Путь до БД

- Задается в `.env` файле
- Выглядит примерно так

 ```
DATABASE_URL="postgresql://postgres:pas123@localhost:5432/indamovie?schema=public"
 ```

- БД надо создать руками, призма сама ее не создает



# Схема, генерация таблиц

## Описание схемы

- Можно пойти двумя путями
  - Генерация таблиц по схеме
  - Привязка существующих таблиц вручную к схеме
- Схему описывать в файле `prisma/schema.prisma`
- Выглядит примерно так

```
generator client {
  provider = "prisma-client"
  output   = "../src/backend/generated/prisma"
}

datasource db {
  provider = "postgresql"
}

model Project {
  id            Int        @id @default(autoincrement())
  code          String     @unique @db.VarChar(50)
  title         String     @db.VarChar(255)
  projectStatus String     @map("project_status") @db.VarChar(50)
  createdAt     DateTime   @default(now()) @map("created_at")
  
  characters Character[]
  scenes     Scene[]
  lines      Line[]

  @@map("project")
}

model Character {
  id               Int      @id @default(autoincrement())
  projectId        Int      @map("project_id")
  fullName         String   @map("full_name") @db.VarChar(255)
  shortName        String?  @map("short_name") @db.VarChar(100)
  portraitFilename String?  @map("portrait_filename") @db.VarChar(255)
  
  project  Project @relation(fields: [projectId], references: [id], onDelete: Cascade)
  lines    Line[]

  @@index([projectId], name: "idx_character_project_id")
  @@map("character")
}

model Scene {
  id               Int      @id @default(autoincrement())
  projectId        Int      @map("project_id")
  pictureFilename  String   @map("picture_filename") @db.VarChar(255)
  
  project Project @relation(fields: [projectId], references: [id], onDelete: Cascade)
  lines   Line[]

  @@map("scene")
}

model Line {
  id              Int     @id @default(autoincrement())
  projectId       Int     @map("project_id")
  lineText        String  @map("line_text") @db.Text
  characterId     Int?    @map("character_id")
  inProjectOrder  Int     @map("in_project_order")
  sceneId         Int?    @map("scene_id")
  inSceneOrder    Int?    @map("in_scene_order")
  
  project   Project   @relation(fields: [projectId], references: [id], onDelete: Cascade)
  scene     Scene?    @relation(fields: [sceneId], references: [id], onDelete: SetNull)
  character Character? @relation(fields: [characterId], references: [id], onDelete: SetNull)

  @@index([projectId], name: "idx_line_project_id")
  @@index([characterId], name: "idx_line_character_id")
  @@map("line")
}
```



## Генерация таблиц

```
npx prisma generate
```

- После этого в БД должны появиться таблицы, соответствующие схеме
- А в директории, заданной через output, - клиент и классы модели



# Простой пример использования клиента

- Где-нибудь создаем файл с инстансом клиента призмы

```typescript
import { PrismaClient } from '@/src/backend/generated/prisma/client';
import { PrismaPg } from '@prisma/adapter-pg';

const adapter = new PrismaPg({ 
  connectionString: process.env.DATABASE_URL
});

export const prisma = new PrismaClient({ adapter });
```

- Пользуемся клиентом

```typescript
import { IProjectRepository } from "@/src/backend/core/repository/IProjectRepository";
import { Project } from "@/src/backend/generated/prisma/client";
import { prisma } from "@/src/backend/infrastructure/prismaClient";

export class PostgresProjectRepository implements IProjectRepository {
  async create(projectCode: string): Promise<Project> {
    return prisma.project.create({
      data: {
        code: projectCode,
        projectStatus: 'draft',
      },
    });
  }
}
```

## Классы моделей

- Они называются с суффиксом Model, например ProjectModel
- Но призма их ре-экспортирует под простыми именами для удобства, например Project
- Просто надо импортировать из правильного места - файла `generated/prisma/client`

```typescript
import { Project } from "@/src/backend/generated/prisma/client";
```
