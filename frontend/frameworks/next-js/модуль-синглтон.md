- Next.js кэширует модули между запросами
- Соответственно, если например нам надо *единожды* создать конфиг, который будет доступен для всех запросов, то
  - мы просто создаем файл, в нем функцию создания конфига, тут же в этом файле ее вызываем, кладем результат в переменную и ее экспортируем
  - потом просто импортируем эту переменную во всех файлах страниц, где она нужна и всё
    - Конфиг создастся при первой загрузке модуля, и т.к. модуль закешируется, переменная с конфигом при последующих запросах будет использована та же самая
