# box-sizing, Алгоритм расчета размера элемента

- Значения `box-sizing`:
  - `border-box` - width для элемента задает сразу его *итоговую ширину* (контент + паддинг + бордер)
  - `content-box` - width для элемента задает *ширину только его контента*
    - К ней прибавляется паддинг, бордер и получается итоговая ширин
- P.S. Для height то же самое правило расчета

Пример:

- `width: 150px`

- `padding: 10px`

- `border: 1px`

При border-box итоговая ширина элемента 150px; размер его контента 150 - 10*2 - 1*2 = 128px.

При content-box, 150px это только ширина контента. Итоговая ширина 150 + 10*2 + 1*2 = 172px.

# Историческая справка

- На практике гораздо удобнее использовать `box-sizing: border-box`
  - Потому что так элемент всегда имеет предсказуемый размер. Если меняется контент, то размер элемента не меняется, стало быть из-за изменившегося элемента не поедет остальная верстка
- Значение `content-box` по умолчанию было признано архитектурной ошибкой css ([источник](https://wiki.csswg.org/ideas/mistakes)):

> Box-sizing should be `border-box` by default.

- Поэтому есть смысл всегда менять это значение на `border-box` для всех элементов:

```css
* {
  box-sizing: border-box;
}
```
