# Особенности массивов

- Массивы имеют фиксированный размер, добавление \ удаление не поддерживают
- Неинициализированные элементы заполняются дефолтным (термин "zero value") для своего типа значением:
  - числа - `0`
  - строки - `""`
  - boolean - `false`



# Особенности слайсов

- Концептуально слайс - это "динамический массив"
  - Т.е. можно добавлять \ удалять элементы

# Создание

## Пустой

Массив:

```go
var langs [5]string  // Размер указать обязательно
```

Слайс:

```go
var langs []string  // Размер не обязателен
```

## Инициализированный

Массив:

```go
langs := [...]string{"golang", "javascript", "typescript", "java", "php"}
```

```go
langs := [5]string{"golang", "javascript", "typescript", "java", "php"}
```

Слайс:

```go
langs := []string{"golang", "javascript", "typescript", "java", "php"}
```







# Обход

- Массивы и слайсы обходятся по одинаковым принципам

## Способы обхода

### Итератором

```go
langs := [...]string{"golang", "javascript", "typescript", "java", "php"}

for ind, item := range langs {
  fmt.Printf("[%d]: %s\n", ind, item)
}
```

- Считается идиоматичным способом обхода
- Дает и доступ к индексу (если надо), и к элементу
- Подходит для большинства сценариев

### "Классика"

```go
langs := [...]string{"golang", "javascript", "typescript", "java", "php"}

for i := len(langs) - 1; i >= 0; i-- {
  fmt.Println(langs[i])
}
```

- Для обычного обхода - не идиоматично
- Лучше использовать только в нестандартных случаях
  - Обход с конца
  - Обход с нестандартным шагом
  - Обход только четных \ нечетных позиций
  - И т.д.
- Дополнительно
  - `len(langs)` не надо выносить перед циклом, это не дает выигрыша
