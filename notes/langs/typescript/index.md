# Типы данных, обзор

## Принципы типизации

- В TypeScript структурная типизация
  - это значит, что типы считаются совместимыми, если у них совпадает набор свойств и методов
    - P.S. не забываем, что методы в JS - это тоже свойства, просто в них лежат функции
  - совпадение не подразумевает полную идентичность в наборе свойств
    - главное чтобы в одном из типов были все свойства из другого
      - +в другом могут быть дополнительные
        - T1[A,B] => T2[A,B,C,D], T1 совместим с T2, но T2 несовместим с T1
        - [Пример совместимости при структурной типизации](типы/принципы-типизации#пример-совместимости-при-структурной-типизации)
  - в языках вроде C# используется *номинальная типизация* (nominal typing)
    - это когда совместимость типов определяется их именем
- Выведение типа
  - [Явное указание типа](типы/принципы-типизации#явное-указание-типа)
    - тип указывается через двоеточие `:` после имени переменной или параметра функции
    - для функции тип указывается после круглых скобок
  - [Автоопределение типа (infer)](типы/принципы-типизации#автоматическое-определение-типа)
    - компилятор может вычислить тип автоматически при присваивании переменной значения
      - поэтому в таких случаях можно не указывать тип
    - если не указать тип и при этом не присвоить значение, то тип будет any
- Типы существуют только в момент компиляции, в рантайме их нет
  - Исключение - enum

# Конфиги, фичи

- Два режима
  - ScriptMode
    - Файлы без import \ export, все содержимое таких файлов доступно глобально
    - `.d.ts` файлы (d - declaration), для глобального объявления \ расширения существующих типов
  - ModuleMode
    - Файлы с import \ export, содержимое таких файлов доступно только через import
- Module Augmentation

- Recursive Types



### **Высокая важность (используются постоянно):**

- **Type guards** - ежедневно для сужения типов
- **Assertion functions** - часто для валидации
- **Mapped Types** - постоянно в utility types и библиотеках
- **Declaration merging** - регулярно в экосистеме (например, расширение Express Request)

### ⚠️ **Средняя важность (полезно знать):**

- **Условные типы** - часто в библиотеках, реже в app-коде
- **Ambient declarations (.d.ts)** - нужно для работы с JS-библиотеками
- **Template Literal Types** - мощно, но niche use cases

### 🔶 **Низкая важность (можно отложить):**

- **Branded types** - продвинутый паттерн для номинальной типизации
- **Recursive types** - редко outside JSON/деревьев
- **namespace/module** - устаревший подход, современные проекты используют ES modules
