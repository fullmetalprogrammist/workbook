# Синтаксис

## Разновидности функций

- FD и FE
  - [FD (**F**unction **D**eclaration)](синтаксис#function-declaration)
    - всплывает
  - [FE (**F**unction **E**xpression)](синтаксис#function-expression)
    - не всплывает
      - потому что создается уже в момент выполнения кода
    - Виды
      - Анонимная
      - Именованная - NFE - **N**amed **F**unction **E**xpression
        - именование нужно, если надо вызывать в функции саму себя
        - кроме как внутри самой функции это имя нигде не доступно
  - Как отличить
    - синтаксис с виду одинаковый, но
      - если функция объявлена в месте, где она является частью выражения (присваивание, тернарник и т.д.), - то это FE
      - если функция является statement'ом, т.е. как бы сама по себе, то это FD
- [Лямбды](синтаксис#лямбды)
  - они же "стрелочные" из-за `=>` в синтаксисе
  - варианты синтаксиса
    - нет параметров \ один параметр \ несколько параметров
    - тело из одной строки \ тело из нескольких строк
- [Функции-конструкторы](синтаксис#функции-конструкторы)
  - подробно о ФК в конспектах про объекты и прототипы

## Параметры функции

- передаются по значению
- undefined, если не передать значение
  - Можно задавать значения по умолчанию
- [arguments](синтаксис#arguments)
  - это псевдомассив
    - в него собираются все аргументы, переданные в функцию
  - технически - это локальная переменная, которая автоматически создается внутри каждой функции
    - 🔥кроме лямбд

## Результат функции

- возвращается через return
- если ничего не возвращать, результат все равно будет - это undefined

## Прочее

- Терминология
  - [Колбэки](синтаксис#колбэки)
    - это просто название для функций, которые мы не вызываем сами, а просто передаем в другую функцию и она сама решает, когда их вызвать



# this

- this это ключевое слово, не переменная и не поле
- момент вычисления this
  - вычисляется каждый раз в момент вызова функции
    - правила вычисления зависят от того как вызвана функция
- правила вычисления
  - ⚡ автоматическое вычисление this
    - 5 сценариев
      - [this в top-level скрипта](автоматическое-вычисление-this#this-в-top-level-скрипта)
        - указывает на глобальный объект (не зависимо от режима, что строгий, что обычный)
      - [this при вызове обычной функции](автоматическое-вычисление-this#this-при-вызове-обычной-функции)
        - обычный режим - указывает на глобальный объект
        - строгий режим - undefined
      - [this при вызове функции как метода объекта](автоматическое-вычисление-this#this-при-вызове-функции-как-метода-объекта)
        - указывает на этот объект
      - [this при вызове функции через new](автоматическое-вычисление-this#this-при-вызове-функции-через-new)
        - указывает на объект, создаваемый неявно в начале вызова функции-конструктора
      - [this в лямбдах](автоматическое-вычисление-this#this-в-лямбдах)
        - this такой же, как во внешнем блоке на момент *создания* лямбды
  - ⚡ явное указание this при вызове функции
    - 2 сценария
      - Методы функции
        - [call, apply](call-apply-bind#call-и-apply)

# Продвинутое

## Функция - это объект

- Функция - это объект
  - технически функция, независимо от способа объявления, является объектом
  - у функции есть свойства и методы
  - свойства, пара полезных
    - `.name` - имя функции
    - `.length` - количество параметров функции
      - rest-параметры не считаются, только "обычные"
  - методы, пара полезных
    - call, apply, bind - все связаны с this, см раздел про this
