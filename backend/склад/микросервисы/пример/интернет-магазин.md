# 1. User Service (покупатели)

**Отвечает за:**

- хранение покупателей
- получение информации о покупателе
- список покупателей

**Данные:**

```
User
id
name
email
```

**Что делает:**

- возвращает данные пользователя
- проверяет существование пользователя

**Чего НЕ делает:**

- не знает ничего о заказах
- не знает ничего о товарах
- не знает ничего об оплате

------

# 2. Product Service (каталог товаров)

**Отвечает за:**

- каталог товаров
- информацию о товаре
- цену товара

**Данные:**

```
Product
id
name
price
description
```

**Что делает:**

- возвращает список товаров
- возвращает данные конкретного товара
- сообщает цену товара

**Чего НЕ делает:**

- не знает кто покупает
- не знает остатки
- не знает заказы

------

# 3. Inventory Service (остатки)

**Отвечает за:**

- количество товара на складе
- резервирование товара
- освобождение резерва

**Данные:**

```
Inventory
productId
availableQuantity
```

**Что делает:**

- сообщает сколько товара доступно
- резервирует товар для заказа
- освобождает резерв

**Чего НЕ делает:**

- не знает покупателей
- не знает оплату
- не знает заказы целиком

------

# 4. Order Service (заказы)

**Главный бизнес-сервис.**

**Отвечает за:**

- создание заказа
- хранение заказов
- статус заказа
- координацию процесса заказа

**Данные:**

```
Order
id
userId
status
totalPrice
OrderItem
productId
quantity
price
```

**Что делает:**

- принимает запрос на создание заказа
- проверяет пользователя
- получает информацию о товарах
- резервирует товар
- инициирует оплату
- сохраняет заказ
- меняет статус заказа

**Чего НЕ делает:**

- не хранит каталог товаров
- не управляет складом
- не проводит оплату

------

# 5. Payment Service (оплата)

**Отвечает за:**

- обработку оплаты
- результат оплаты

**Данные (минимально):**

```
Payment
orderId
amount
status
```

**Что делает:**

- принимает запрос на оплату
- возвращает результат оплаты

**Чего НЕ делает:**

- не знает товары
- не знает склад
- не создаёт заказы

------

# 6. Notification Service (уведомления)

**Отвечает за:**

- отправку уведомлений пользователю

**Что делает:**

- отправляет сообщение пользователю
- логирует уведомления

**Примеры уведомлений:**

```
Order created
Payment successful
Payment failed
```

**Чего НЕ делает:**

- не управляет заказами
- не знает товары
- не делает оплату
