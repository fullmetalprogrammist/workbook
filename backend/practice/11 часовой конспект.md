# Рефакторинг

- Программа не пишется сразу идеальной
  - Она пишется сперва так чтобы быть просто адекватной текущей ситуации, а потом рефакторится по мере поступления новой информации



# SRP

- Начинать писать функцию можно без SRP, но с заделом на будущее разделение
  - Для этого не смешиваем внутри функции реализацию множества вещей, а разделяем на блоки
  - Когда их становится достаточно для того, чтобы функцию стало тяжело читать, разделяем на отдельные функции
  - Полученные части вызываем в объединяющей функции
